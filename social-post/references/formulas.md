# 13 個貼文公式（小帳號可複製）

篩選後的 viral 公式。**過濾掉**：名人/大號（>100K 粉）、付費廣告、MLM/博弈、HubSpot 式泛泛 tips。

選公式看三件事：(1) 今天的**目標**（社群/工坊/KOL/聯盟/**突破鐵粉圈**）(2) **平台**（Threads 吃短句高回覆、FB 吃長敘事、IG 吃圖、X 吃 thread）(3) **當前觸及瓶頸**（若連續多篇 > 85% 追蹤者觸及，必用 Part 2 廣推公式）。

每個公式給：**骨架 / 例 / 為何有效 / 複製度 1-5 / 主攻平台**。

---

## 🧠 FB 2026 演算法廣推訊號（寫公式前必讀）

Meta 2024-11 推「Unconnected Reach」正式指標。2025 Q2 起演算法以**第 1 小時「陌生人分享 + 深度留言 + 停留秒數」**三訊號決定是否跨粉絲圈：

| 信號 | 權重 |
|---|---|
| 分享 | **20× 讚** |
| 留言 | **5× 讚** |
| Dwell time（查看更多點擊 + 圖放大 + revisit）| **隱形王牌** |
| 讚 | 幾乎無權重 |

**Part 1（F1-F7）** 多數訴求「讚+留言」，適合既有粉絲深化。
**Part 2（F8-F13）** 專攻「分享 + dwell time」，能讓演算法跨粉絲圈推廣。

# Part 1：基礎公式（F1-F7）

## F1｜Day-N 開發日誌

**骨架**：`Day N 做＜專案＞：今天解決了＜具體問題＞，卡了＜時間＞在＜坑＞。明天要＜下一步＞。`
**例**：Day 6 做 Claude Skill：今天把邀請碼自動派發跑通了，卡了兩小時在 contenteditable 沒焦點。明天接 Threads 自動發文。
**為何**：連載有追劇效應，第 N 篇吃第 1 篇累積的讀者。小號靠累積不靠爆。
**複製度**：5/5
**平台**：Threads（主）、X

---

## F2｜截圖先丟再講

**骨架**：`[大圖/截圖] + 一句話標題 + 2-3 行解釋 + 一句 CTA`
**例**：Claude 跟我說「這段 code 我拒絕寫」我第一次看到 AI 罷工，草。[截圖]
**為何**：演算法看 media dwell time。截圖 = 免費 dwell time；zoom/save 會把你推到 Explore。
**複製度**：5/5
**平台**：FB、Threads、IG

---

## F3｜實測翻車版

**骨架**：`我用＜工具＞＜N＞天，＜正向結果＞。但有三件事沒人告訴你：1)... 2)... 3)...`
**例**：我用 Claude Code 跑社群自動化 30 天，接了 5 個案。但有三件事沒人告訴你：1) skill 一多 context 就爆 2) Chrome MCP 三天 FB 改版一次 3) Max 流量半天見底。
**為何**：純實測已 fatigue，「翻車/沒人說」反轉能撐。save 率約純誇獎文 2 倍。
**複製度**：4/5
**平台**：FB 長文、X thread

---

## F4｜社群里程碑 + 投票

**骨架**：`＜社群＞＜人數＞！！！＜時間敘述＞，沒想到＜情緒＞。留言告訴我想看 A)... B)... C)...，票最多的＜行動＞。`
**例**：Claude Code 破 1200 人！！！才一個月，沒想到。留言想看 A) Skill 開發 B) Chrome MCP C) 社群運營，票最多的週五開工坊。
**為何**：慶祝+選擇一拍兩響。留言權重 > 讚，小號靠留言上分。
**限制**：實證觸及仍限鐵粉圈（2026-04-24 Day 4 實測 93.9% 追蹤者觸及）。**想擴散用 F13 取代**。
**複製度**：5/5
**平台**：FB、Threads

## F5｜工具對打

**骨架**：`＜A＞ vs ＜B＞ 跑＜具體任務＞。結論：＜洞見＞。細節：[三行對比]`
**例**：Cursor vs Claude Code 跑 skill 重構。結論：Cursor 改得快、Claude Code 想得深。我早上 Cursor 衝刺、下午 Claude Code 收尾。
**為何**：對比文 save 率 3-5 倍。演算法搜工具名時會 surface 給非追蹤者。
**複製度**：5/5
**平台**：X thread（主）、Threads

### F6b：meta-ship 變體（2026-04-21 + 2026-04-28 雙實證 mega-viral）

**鐵則：4 段 4 句**（每段 = 1 個 sentence，段間空行分隔，**不是螢幕視覺 4 行字**）

```
[段 1：成果炸場 + meta 鉤子] ＜工具/AI＞這次真的殺瘋了，我直接做了一個＜產品＞能＜能力敘述＞！！！你沒看錯！！＜自證懸念，例：這篇貼文就是它自己發的＞！！真的太神啦！！！

[段 2：過程數字] 剛剛花＜時間＞讓它＜具體動作＞，＜量化成果＞

[段 3：使用體驗敘事] 現在我說「＜指令＞」它就＜動作＞

[段 4：社群召集 CTA（非邀請碼）] ＜未來計畫＞，沒入群的留言我拉你
```

**結構強制檢查**：
- ！！！只在段 1 密集（後三段純敘述語氣）
- 段 2/3 結尾不加標點不加 emoji
- 段 4 結尾留白（無 emoji 無標點）
- 絕不用 numbered / bullet list
- 剛好 4 段不多不少（5 段 = 破節奏；3 段 = 缺 punch）

#### F6b 三種敘事變體（v0.6 新增變體 D）

**變體 A：promo 型（self-evident 鉤子）**
- meta 鉤子 = 「這篇 po 就是它自己發的」（產品自證）
- 用於 shipping announcement（新東西第一次見光）
- 實證 Day 1（2026-04-21）：瀏覽 75,071 人 / 380 讚 / 457 留言 / **81 分享** / +167 FB 粉 / **+1,319 Line 社群成員** / 96.5% 非追蹤者 / 排名 1/10 / 6 天 plateau 仍漲

**變體 B：演算法復盤型（self-experimental 鉤子）**
- meta 鉤子 = 「跑了 N 篇對照組打臉自己以為對的事」（實驗自證）
- 用於 launch 後的 retro / lesson-sharing
- 實證 Day 6（2026-04-28，**接 Day 1 後 7 天**）：26h plateau 18,603 觀眾 / 100 讚 / 89 留言 / 25 分享 / **92.6% 非追蹤者** / 排名 2/10

**變體 D：社群 social proof 型（self-evident growth 鉤子）** 🆕 v0.6
- meta 鉤子 = 「我帶起來的社群現在 N 人，全是 X」（成員 social proof + 成長軌跡）
- 用於 pitch 社群價值 / 拉群 funnel
- 實證 5/5（2026-05-05，**接 Day 7 後 6 天 / 接 Day 1 後 14 天**）：19h 44,110 觀眾 / 265 讚 / 342 留言 / 35 分享 / **95.3% 非追蹤者** / **+1,239 Line 群成員**（→ 3,575）/ 連結點擊 1（全靠留言區私訊拉群）

```
[段 1：成果炸場+meta 鉤子] ＜AI＞這次真的殺瘋了，我用這個 skill 帶起來的＜社群＞現在＜人數＞，＜社群價值描述＞！！！你沒看錯！！＜成員 social proof＞！！真的太神啦！！！

[段 2：過程數字] ＜時間前＞還＜舊人數＞，發＜貼文數＞貼文衝到＜新人數＞，＜流量轉化機制＞

[段 3：使用體驗敘事] 現在我問「＜情境問題＞」群裡＜時間＞就有人＜動作＞，比＜對照物＞還快

[段 4：社群召集 CTA] ＜社群名＞歡迎一起來，沒入群的留言我拉你
```

**變體 D 跟 A/B 的差異**：
- 變體 A 推產品（「我做了 X」）
- 變體 B 推洞察（「我學到了 X」）
- 變體 D 推**社群本身**（「我帶起的社群長這樣」）— social proof 比直接賣 hard sell 更耐燒

**跳過變體 C**：Day 7 商業 ROI 嘗試 fail，不算成功變體。

**為何**：
- **meta 鉤子**引發 curiosity，觸發分享（不論是 self-evident 還 self-experimental）
- **留言 CTA 取代邀請碼 CTA**：更乾淨、更導向社群
- 兩變體互補：變體 A 拉新粉，變體 B 拉討論深度 + 突破鐵粉圈

**⚠️ 重要衰減警告**（v0.6 校正版）：「殺瘋了」hype 開頭對**非追蹤者**有 40% 衰減**僅當敘事意圖重複時**。5/5 用 hype 開頭 + 換到「社群 social proof」敘事意圖（月內第 5 個 F6b）→ 仍 95.3% 非追蹤者 + 44K 觀眾。**衰減主因是「敘事意圖重複」，不是 hype 詞語、不是 F6b 公式總配額、不是 14 天間隔**。

**複製度**：5/5（變體 A/B/D 交替可無限期持續，每 4 天就能再爆，只要主題庫還有新意圖）
**平台**：FB（主戰場）、IG（配圖時）

### 何時用 F6a vs F6b
- **用 F6a**：推廣**他人的工具**、有邀請碼或 referral 制度
- **用 F6b**：推廣**自己做的東西** + 復盤 + 社群 pitch + N 個其他變體

### F6b 變體間隔建議（v0.6 簡化）
- **同變體**：4 天冷卻 + 月內 ≤ 2 次（v0.5 的「14 天間隔」不必要）
- **跨變體**：完全不限頻率（5/5 距 Day 1 14 天距 Day 7 6 天，新變體 D → mega-viral）
- **連發 F6b 沒問題**只要每篇敘事意圖不同（v0.5 R9「月配額硬限」假設推翻）

# Part 2：廣推公式（突破鐵粉圈，F8-F13）

**使用時機**：最近 3 篇 > 85% 追蹤者觸及、非追蹤者 < 15% → 演算法把你歸類為「鐵粉限定創作者」，必須用這些公式強制擴散。

## F8｜Credibility Piggyback（外部實體標記挑戰）

**骨架**：`我剛用 ＜工具/框架＞ ＜具體反直覺動作＞，＜量化結果＞。@＜官方/開發者＞ 你們下一版能不能＜具體建議＞？`
**例**：剛用 Claude Skill 讓它自己幫自己寫 skill，跑了 14 篇貼文 0 bug。@Anthropic 可以讓 skill 互相 call 嗎？真的會爆。
**為何廣推**：標記**有名字的外部實體**（@Anthropic / @OiiOii / @某 dev），演算法把貼文**同時推給「追蹤該實體」的人** = cross-audience seeding。對方員工或粉絲一留言，陌生人觸及比例暴漲。
**跟 F6b 差別**：F6b meta 只能用一次，**F8 每次挑不同對象都是新觸發源**，可持續。
**限制**：tag 必須真實且具體相關（tag 錯對象或亂 tag = 降信任）。
**複製度**：5/5
**平台**：FB、X、Threads

---

## F9｜Public L-taking（我錯了反轉文）

**骨架**：`三個月前我說＜過去堅定觀點＞，我錯了。今天＜具體事件＞讓我重新想：＜新觀點 3 行＞。`
**例**：三個月前我說「AI 不可能取代社群經理」，我錯了。昨天 skill 自己回了 40 則留言還拉 5 個人進群，我只是在旁邊看。現在我的工作是設計 skill 的個性，不是打字。
**為何廣推**：
- 承認錯誤 = 強烈情緒觸發
- **超高 dwell time**（讀者停下來想看到底錯在哪）
- 分享率特別高 —— 「轉給認為這議題有趣的朋友」比轉正向炫耀文心理阻力低 3 倍
**跟 F3 差別**：F3「翻車」吐槽**工具**，F9 是**吐槽自己**，更脆弱感、更易被長輩/同儕分享。
**複製度**：3/5（需要真實錯誤案例）
**平台**：FB 長文

## F10｜Controversy Middle Ground（兩派戰爭居中開炮）

**骨架**：`＜A 派＞說＜立場＞，＜B 派＞說＜立場＞，兩邊都錯。真相是＜你的第三立場＞。＜2-3 行論證＞。`
**例**：Vibe coder 說 AI 要取代工程師，老工程師說 AI 寫的都垃圾，兩邊都錯。真相是：AI 取代的不是寫 code 的人，是「只會寫 code」的人。會設計 context、會命名、會拒絕爛需求的，反而變 10x。
**為何廣推**：兩派支持者都會留言反駁/支持 → 留言數暴衝 → 演算法判為「爆點討論」強推陌生人。**居中比選邊風險低**：不會被單邊取消。35-44 歲受眾吃「論理清晰的第三條路」不吃網戰。
**限制**：**每週 1 次上限**，超過會被觀眾貼標「喜歡煽動」。
**複製度**：4/5
**平台**：FB、X thread

## F11｜Counter-Funnel Giveaway（反漏斗教學）

**骨架**：`今天免費送＜原本私藏的東西＞：[內容 or 連結]。不用追蹤不用留言，拿走就好。只有一個條件：＜低門檻反 CTA＞。`
**例**：把我這個月跑爆款的 skill 整包開源在 GitHub（連結）。不用追蹤我、不用按讚，下載就好。只拜託一件事：你改進後回我一份，我們一起優化。
**為何廣推**：
- **反 CTA** 觸發「這人不是在收割」信任感 → 自發分享率高
- Meta AI 2025 新增「perceived authenticity score」，反收割姿態直接吃這個加分
- 「推薦一個不割韭菜的」是 35-44 歲最愛轉的文
**限制**：**每月上限 1 次**，太常用會被識破。
**跟 F6 差別**：F6 有 CTA（邀請碼、拉群），F11 **沒有任何對自己的好處訊號**，看似吃虧反而極致擴散。
**複製度**：3/5
**平台**：FB、Threads

## F12｜Timestamp Live-ops（時間戳 + 過程直播）

**骨架**：`[精確時間] 我現在正在＜具體行動＞，目標是＜量化目標＞，＜N＞小時後回來報結果。＜1 行懸念＞。`
**例**：現在 22:47，我要在 3 小時內用 Claude Code 跑完 5 個 skill 的整合測試。如果成功，明早 skill 生態就有第一個「skill of skills」。失敗就開源當反面教材。
**為何廣推**：
- 時間戳 = **重返訪問訊號**（讀者回來看結果）→ 單一貼文 revisit 率破 30% 直接觸發 FB「高價值內容」判定
- 留言「追蹤結果」本身就是留言權重
- 比一次性「我做完了」回顧擴散強 2-3 倍
**後續配對**：發「結果」那篇時**引用原貼文**，演算法把兩篇串連推給看過第一篇的陌生人。
**跟 F1 差別**：F1 Day-N 是已完成的回顧，**F12 是未來式懸念**，緊迫感強得多。
**複製度**：4/5
**平台**：FB、Threads

## F13｜Named Gratitude Chain（具名致謝鍊結）

**骨架**：`這件事沒有 ＜3-5 個人名＞ 就不會發生：@A ＜具體貢獻 1 行＞、@B ＜貢獻 1 行＞、@C ＜貢獻 1 行＞。＜1 句總結＞。`
**例**：Claude Code 社群過 4000 人，沒有這幾個人就沒有今天：@游一龍 第一個幫我測 skill 到天亮、@Susan Fu 設計了社群視覺、@林于勝 每週回報 bug。我只是把他們的努力放在同一個舞台。
**為何廣推**：
- 每個被 tag 的人**自動觸發通知+高機率分享回自己的牆** → 每 tag 1 人等於多一個漏斗口
- 5 個 tag 平均覆蓋 5-15K 非追蹤者
- 比 F4 純里程碑擴散 5 倍
**限制**：**必須真實且具體**，空泛「感謝大家」沒效。**每月 1 次上限**。
**跟 F4 差別**：F4 是「人數+投票」，F13 是「人名+貢獻」，更溫度、更難被演算法判為 self-promo。
**複製度**：4/5（需要真實貢獻者 + 他們願被 tag）
**平台**：FB（主）、Threads

# Part 3：分享驅動公式（thought leader 圈，F14-F17）

**使用時機**：v0.6 後新增。當你想拓展到「professional thinker 圈」（PM / 設計師 / dev tool 創業者）= 跟 Hao F6b 留言區拉群 funnel 互補的「FB 追蹤 + 分享 + GitHub star」funnel。

**主要差異**：F1-F13 是讚 / 留言驅動，F14-F17 是**分享驅動**（v0.6 演算法權重 20× 讚）。

**Hao 適配度評分**：

| 公式 | 寫作時間 | 等效讚分 | 適配度 | 適用時機 |
|---|---|---|---|---|
| F14 演講濃縮型 | 3-5 h | ~21K | ⭐⭐ | 看完一場好演講 / 一本書 |

## 2026 高停留開頭（換掉失效的 hype 開頭）

35-44 歲 AI 受眾實測高停留率前 5 開頭：

1. **數字 + 動詞矛盾**：「4,180 個追蹤者，昨天有 72,000 人看。」
2. **時間戳 + 事件**：「2026-04-21 凌晨 2 點，我按下發佈那一秒，我沒想到⋯⋯」
3. **反問具體化**：「如果你的 AI 會拒絕幫你寫 code，你會生氣還是感動？」
4. **公開認錯**：「三個月前我說過的話，我想收回。」
5. **金額/時間成本**：「這件事花了我 $47 鎂 + 6 小時，給你省掉兩個。」

**失效開頭（觸及衰減 40%+）**：
- 「這次真的殺瘋了」
- 「各位！！！」
- 「真的太神啦」類 hype 開頭
- 「我又有新的 XX 了」

**注意**：失效開頭對**既有粉絲**仍有效（鐵粉買帳），只對**非追蹤者**衰減。想擴散一定要換新開頭。
